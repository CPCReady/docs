## Descripción General

**CPCReady** es una herramienta basada en Python para gestionar imágenes de disco Amstrad CPC. Sigue esta guía para instalarla en tu sistema.

<Aside type="note">
CPCReady funciona en **macOS**, **Linux** y **Windows** (vía WSL). Python 3.13+
</Aside>

## Métodos de Instalación

<Tabs>
  <TabItem label="Desde Código Fuente (Recomendado)">

### Instalar desde Repositorio GitHub

<Steps>

1. **Clonar el repositorio**
   
   ```bash
   git clone https://github.com/CPCReady/cpc.git
   cd cpc
   ```

2. **Instalar con Poetry**
   
   ```bash
   poetry install
   ```

3. **Verificar instalación**
   
   ```bash
   poetry run cpc --version
   ```

4. **Opcional: Añadir al PATH**
   
   Para un acceso más fácil, puedes crear un alias en tu shell:
   
   ```bash
   # Para bash/zsh, añadir a ~/.bashrc o ~/.zshrc
   alias cpc='poetry run cpc'
   ```

</Steps>

<Aside type="tip">
Instalar desde código fuente te da las últimas características y te permite contribuir al desarrollo.
</Aside>

  </TabItem>
  
  <TabItem label="Desde PyPI (Futuro)">

### Instalar desde Índice de Paquetes Python

<Aside type="caution">
El paquete PyPI aún no está disponible. Usa el método "Desde Código Fuente" por ahora.
</Aside>

Cuando esté disponible, podrás instalar con:

```bash
pip install cpcready
```

Verificar instalación:

```bash
cpc --version
```

Actualizar a la última versión:

```bash
pip install --upgrade cpcready
```

  </TabItem>
</Tabs>

## Configuración Post-Instalación

### 1. Configurar CPCReady

Ejecutar el asistente de configuración:

```bash
poetry run cpc configweb
```

O editar manualmente el archivo de configuración:

**Ubicación**: `~/.config/cpcready/cpcready.toml`

```toml
[drive]
drive_a = ""
drive_b = ""
selected_drive = "A"

[emulator]
default = "RetroVirtualMachine"
retro_virtual_machine_path = "/Applications/Retro Virtual Machine 2.app"

[system]
user = 0
model = "6128"
mode = 1
```

### 2. Configurar Emulador (Opcional)

Si quieres usar el `cpc run` comando, configura la ruta de tu emulador:

<Steps>

1. Descargar **RetroVirtualMachine** desde [retrovirtualmachine.org](https://www.retrovirtualmachine.org/)

2. Instala el emulador en tu sistema

3. Configura la ruta en CPCReady:
   
   ```bash
   poetry run cpc configweb
   ```
   
   O editar `~/.config/cpcready/cpcready.toml` directamente:
   
   <Tabs>
     <TabItem label="macOS">
     ```toml
     [emulator]
     retro_virtual_machine_path = "/Applications/Retro Virtual Machine 2.app"
     ```
     </TabItem>
     
     <TabItem label="Linux">
     ```toml
     [emulator]
     retro_virtual_machine_path = "/usr/local/bin/RetroVirtualMachine"
     ```
     </TabItem>
     
     <TabItem label="Windows (WSL)">
     ```toml
     [emulator]
     retro_virtual_machine_path = "/mnt/c/Program Files/RetroVirtualMachine/rvm.exe"
     ```
     </TabItem>
   </Tabs>

</Steps>

## Verificar Instalación

Prueba tu instalación con estos comandos:

```bash
# Verificar versión
poetry run cpc --version

# Ver ayuda
poetry run cpc --help

# Crear un disco de prueba
poetry run cpc disc new test.dsk

# Montar el disco
poetry run cpc drive A test.dsk

# Listar archivos (debería estar vacío)
poetry run cpc cat
```

<Aside type="tip" title="¡Éxito!">
Si todos los comandos se ejecutan sin errores, CPCReady está correctamente instalado y listo para usar.
</Aside>

## Actualizar CPCReady

Para actualizar a la última versión:

```bash
cd /path/to/cpc
git pull origin main
poetry install
```

## Solución de Problemas

### Poetry no encontrado

Si `poetry` no se encuentra, añádelo a tu PATH:

```bash
export PATH="$HOME/.local/bin:$PATH"
```

Añade esta línea a tu `~/.bashrc` o `~/.zshrc` para hacerlo permanente.

### Problemas con la versión de Python

Asegúrate de tener Python 3.13 o superior:

```bash
python3 --version
```

Si es necesario, instala una versión más reciente desde [python.org](https://www.python.org/downloads/).

### Errores de permisos

En Linux/macOS, puede que necesites ejecutar con los permisos apropiados:

```bash
sudo poetry install
```

### Problemas específicos de WSL

En Windows con WSL, asegúrate de usar la versión Linux de Python, no la versión Windows.
