## Overview

**CPCReady** is a Python-based toolchain for managing Amstrad CPC disk images. Follow this guide to install it on your system.

<Aside type="note">
CPCReady works on **macOS**, **Linux**, and **Windows** (via WSL). Python 3.13+
</Aside>

## Installation Methods

<Tabs>
  <TabItem label="From Source (Recommended)">

### Install from GitHub Repository

<Steps>

1. **Clone the repository**
   
   ```bash
   git clone https://github.com/CPCReady/cpc.git
   cd cpc
   ```

2. **Install with Poetry**
   
   ```bash
   poetry install
   ```

3. **Verify installation**
   
   ```bash
   poetry run cpc --version
   ```

4. **Optional: Add to PATH**
   
   For easier access, you can create an alias in your shell:
   
   ```bash
   # For bash/zsh, add to ~/.bashrc or ~/.zshrc
   alias cpc='poetry run cpc'
   ```

</Steps>

<Aside type="tip">
Installing from source gives you the latest features and allows you to contribute to development.
</Aside>

  </TabItem>
  
  <TabItem label="From PyPI (Future)">

### Install from Python Package Index

<Aside type="caution">
PyPI package is not yet available. Use the "From Source" method for now.
</Aside>

When available, you'll be able to install with:

```bash
pip install cpcready
```

Verify installation:

```bash

```

Update to latest version:

```bash
pip install --upgrade cpcready
```

  </TabItem>
</Tabs>

## Post-Installation Setup

### 1. Configure CPCReady

Run the configuration wizard:

```bash
poetry run cpc configweb
```

Or manually edit the configuration file:

**Location**: `~/.config/cpcready/cpcready.toml`

```toml
[drive]
drive_a = ""
drive_b = ""
selected_drive = "A"

[emulator]
default = "RetroVirtualMachine"
retro_virtual_machine_path = "/Applications/Retro Virtual Machine 2.app"

[system]
user = 0
model = "6128"
mode = 1
```

### 2. Set Up Emulator (Optional)

If you want to use the `cpc run` command, configure your emulator path:

<Steps>

1. Download **RetroVirtualMachine** from [retrovirtualmachine.org](https://www.retrovirtualmachine.org/)

2. Install the emulator on your system

3. Configure the path in CPCReady:
   
   ```bash
   poetry run cpc configweb
   ```
   
   Or edit `~/.config/cpcready/cpcready.toml` directly:
   
   <Tabs>
     <TabItem label="macOS">
     ```toml
     [emulator]
     retro_virtual_machine_path = "/Applications/Retro Virtual Machine 2.app"
     ```
     </TabItem>
     
     <TabItem label="Linux">
     ```toml
     [emulator]
     retro_virtual_machine_path = "/usr/local/bin/RetroVirtualMachine"
     ```
     </TabItem>
     
     <TabItem label="Windows (WSL)">
     ```toml
     [emulator]
     retro_virtual_machine_path = "/mnt/c/Program Files/RetroVirtualMachine/rvm.exe"
     ```
     </TabItem>
   </Tabs>

</Steps>

## Verify Installation

Test your installation with these commands:

```bash
# Check version
poetry run cpc --version

# View help
poetry run cpc --help

# Create a test disk
poetry run cpc disc new test.dsk

# Mount the disk
poetry run cpc drive A test.dsk

# List files (should be empty)
poetry run cpc cat
```

<Aside type="tip" title="Success!">
If all commands run without errors, CPCReady is correctly installed and ready to use.
</Aside>

```bash
cd /path/to/cpc
git pull origin main
poetry install
```

## Troubleshooting

### Poetry not found

If `poetry` command is not found, add it to your PATH:

```bash
export PATH="$HOME/.local/bin:$PATH"
```

Add this line to your `~/.bashrc` or `~/.zshrc` to make it permanent.

### Python version issues

Ensure you have Python 3.13 or higher:

```bash
python3 --version
```

If needed, install a newer version from [python.org](https://www.python.org/downloads/).

### Permission errors

On Linux/macOS, you may need to run with appropriate permissions:

```bash
sudo poetry install
```

### WSL-specific issues

On Windows with WSL, ensure you're using the Linux version of Python, not the Windows version.
