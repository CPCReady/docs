## What is CPCReady?

**CPCReady** is a comprehensive command-line toolchain designed for Amstrad CPC enthusiasts and retro computing developers. It provides modern tools to manage disk images, files, and emulator integration from your contemporary development environment.

Built with **Python 3.13+** and powered by **Poetry**, CPCReady brings professional development workflows to vintage computing.

<Aside type="tip" title="Perfect for">
Retro computing enthusiasts, game developers, BASIC programmers, and anyone working with Amstrad CPC disk images and emulators.
</Aside>

## Core Features

<CardGrid stagger>
  <Card title="Disk Management" icon="document">
    Create, format, and manage **.DSK** disk images with full CP/M compatibility. Support for 40 tracks, 9 sectors, 180KB standard format.
  </Card>
  
  <Card title="Virtual Drives" icon="setting">
    Simulate dual floppy drives (A/B) with instant disk insertion, ejection, and status monitoring. Just like the real hardware.
  </Card>
  
  <Card title="File Operations" icon="add-document">
    Save, extract, rename, delete, and list files on disk images. Full support for AMSDOS headers and file types (BASIC, Binary, ASCII).
  </Card>
  
  <Card title="Emulator Integration" icon="rocket">
    Launch **RetroVirtualMachine** with automatic configuration. Supports macOS .app bundles, process management, and file execution.
  </Card>
  
  <Card title="CP/M User Areas" icon="puzzle">
    Authentic CP/M user area support (0-15) for organizing files, just like on real CPC systems.
  </Card>
  
  <Card title="CPC Configuration" icon="laptop">
    Configure CPC model (464/664/6128), video mode (0/1/2), and system settings. Includes GUI configuration tool with **PySide6**.
  </Card>
</CardGrid>

## Quick Start

<Tabs>
  <TabItem label="Basic Usage">

### Essential Commands

```bash
# Mount a disk in drive A
cpc drive A games.dsk

# List files on current disk
cpc cat

# Save a binary file with addresses
cpc save loader.bin b 0x4000 0x4000

# Extract a file from disk
cpc filextr GAME.BAS

# Launch emulator with current disk
cpc run
```

### File Types

<ul>
  <li><strong><code>b</code></strong> - Binary file (requires load/exec addresses)</li>
  <li><strong><code>p</code></strong> - Program file (preserves AMSDOS header)</li>
  <li><strong><code>a</code></strong> - ASCII/text file (no header)</li>
  <li><strong><code>auto</code></strong> - Automatic detection (default)</li>
</ul>

  </TabItem>
  
  <TabItem label="Configuration">

### Configuration File

CPCReady stores settings in TOML format:

**Location**: `~/.config/cpcready/cpcready.toml`

```toml
[drive]
drive_a = "/path/to/system.dsk"
drive_b = "/path/to/games.dsk"
selected_drive = "A"

[emulator]
default = "RetroVirtualMachine"
retro_virtual_machine_path = "/Applications/Retro Virtual Machine 2.app"

[system]
user = 0        # CP/M user area (0-15)
model = "6128"  # CPC model: 464, 664, or 6128
mode = 1        # Video mode: 0, 1, or 2
```

### GUI Configuration

Launch the graphical configuration tool:

```bash
cpc configweb
```

## Command Categories

<CardGrid>
  <Card title="Disk Operations" icon="document">
    **`disc`** - Create, insert, and inspect disk images
    
    - `disc new` - Create blank DSK
    - `disc insert` - Insert DSK into drive
    - `disc info` - Show disk information
  </Card>
  
  <Card title="Drive Management" icon="setting">
    **`drive`** - Manage virtual floppy drives
    
    - `drive A/B` - Mount disk
    - `drive status` - Show drive status
    - `drive eject` - Eject disk
  </Card>
  
  <Card title="File Management" icon="add-document">
    **File operations** on disk images
    
    - `cat` - List all files
    - `save` - Save file to disk
    - `filextr` - Extract file
    - `era` - Delete file
    - `ren` - Rename file
    - `list` - Display BASIC program
  </Card>
  
  <Card title="System & Emulator" icon="rocket">
    **Configuration and execution**
    
    - `run` - Launch emulator
    - `model` - Set CPC model
    - `mode` - Set video mode
    - `user` - Set CP/M user area
    - `settings` - Show configuration
    - `configweb` - GUI configuration
  </Card>
</CardGrid>

## Example Workflow

Here's a typical development workflow with CPCReady:

```bash
# 1. Create a new disk
cpc disc new myproject.dsk

# 2. Mount it in drive A
cpc drive A myproject.dsk

# 3. Save your BASIC program
cpc save menu.bas a

# 4. Save a binary loader
cpc save loader.bin b 0x4000 0x4000

# 5. List files to verify
cpc cat

# 6. Launch in emulator
cpc run MENU.BAS
```
