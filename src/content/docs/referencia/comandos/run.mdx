<Card>
Lanza el emulador previamente seleccionado desde <Badge text="cpc emu" variant="note" /> con el disco seleccionado y ejecuta el archivo especificado. 
Use <Badge text="-A" variant="note" /> or <Badge text="-B" variant="note" /> para especificar la unidad. Si no se especifica una unidad, se utiliza la unidad actualmente seleccionada.
<Aside type="tip">
La unidad seleccionada se configura con <Badge text="cpc drive A" variant="note" /> o <Badge text="cpc drive B" variant="note" />. Por defecto es la unidad A.
</Aside>
</Card>

## Uso
<Aside type="caution">
Actualmente, la funcionalidad de emulación de M4Board está en desarrollo y NO es funcional.
</Aside>
```bash
Uso: cpc run [OPTIONS] [FILE_TO_RUN]

  Run a file from the selected drive in RetroVirtualMachine.

  FILE_TO_RUN: Name of the file to execute from the disk (e.g., DISC,
  GAME.BAS)

  Si -A or -B is not specified, uses the currently selected drive. The file
  must exist in the disk of the selected drive.

Opciones:
  -A, --drive-a  Use disk from drive A
  -B, --drive-b  Use disk from drive B
  --help         Mostrar this message and exit.
```

## Ejemplos

### Lanza el emulador con el disco actual
```bash
cpc run
```

### Ejecutar un archivo específico desde el disco actual
```bash
cpc run DISC.BIN
```

### Ejecutar desde la unidad A
```bash
cpc run GAME.BAS -A
```

### Ejecutar desde la unidad B
```bash
cpc run MENU -B
```

<Aside type="tip">
La ruta del emulador y el modelo CPC deben configurarse primero usando <Badge text="cpc emu" variant="note" /> y <Badge text="cpc model" variant="note" />.
</Aside>

<Aside type="note">
El comando cierra automáticamente cualquier instancia previa de RetroVirtualMachine antes de lanzar una nueva.
</Aside>
