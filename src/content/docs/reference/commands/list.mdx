<Card>Display the contents of BASIC files stored on DSK disk images. Supports tokenized and ASCII BASIC formats with syntax highlighting.</Card>

## Usage

```bash
Usage: cpc list [OPTIONS] FILE_NAME

  List BASIC file from virtual disc.

Arguments:
  FILE_NAME  Name of BASIC file to display

Options:
  -A, --drive-a  Read from drive A (default)
  -B, --drive-b  Read from drive B
  --help         Show this message and exit.
```

## Features

- **Tokenized BASIC**: Automatically detokenizes Locomotive BASIC files
- **ASCII BASIC**: Displays plain text BASIC files
- **Syntax Highlighting**: Rich console output with color coding
- **Format Detection**: Auto-detects BASIC format (tokenized vs ASCII)
- **Binary Detection**: Warns if file is binary (not BASIC)

## Examples

### List BASIC file from drive A
```bash
cpc list GAME.BAS
```

### List BASIC file from drive B
```bash
cpc list MENU.BAS -B
```

### Alternative command
```bash
cpclist PROGRAM.BAS
```

### Display tokenized BASIC
```bash
cpc list LOADER.BAS
```

<Aside type="tip">
The command automatically detects the BASIC format:
- **Tokenized**: Binary format with BASIC tokens (most CPC programs)
- **ASCII**: Plain text format created with editors or `SAVE "FILE",A`
</Aside>

<Aside type="caution">
If you try to list a binary file (type 2), the command will display an error message:
```
Error: FILE.BIN is a BINARY file (type 2), not a BASIC program.
Use 'cpc filextr' to extract binary files.
```
</Aside>

<Aside type="note">
The command respects the current user number (see `cpc user`). Only files in the active user area will be accessible.
</Aside>
