## Subcommands

### rvm status

Check RetroVirtualMachine installation and version compatibility.

```bash
Usage: cpc rvm status [OPTIONS]

  Check RetroVirtualMachine installation and version.

  Verifies:
  - RVM path is configured
  - RVM executable exists at the configured path
  - RVM version matches the required version

Options:
  --help  Show this message and exit.
```

### rvm config

Configure RetroVirtualMachine executable path with interactive prompt and validation.

```bash
Usage: cpc rvm config [OPTIONS]

  Configure RetroVirtualMachine executable path.

  Interactive prompt to set the path to RetroVirtualMachine.

  Validates that:
  - The path exists
  - The executable is accessible
  - The version matches the required version

Options:
  --help  Show this message and exit.
```

## Required Version

<Aside type="caution" title="Version Requirement">
CPCReady requires **exactly** this version of RetroVirtualMachine:

- **Version:** Retro Virtual Machine v2.0 BETA-1 r7
- **Build:** MacOs x64 Build: 6783 - (Tue Jul  9 18:18:12 2019 UTC)

Other versions may not work correctly with CPCReady's automation features.
</Aside>

## Platform Support

### macOS
- **Format:** `.app` bundle (e.g., `/Applications/Retro Virtual Machine 2.app`)
- **Validation:** Locates the executable binary inside `Contents/MacOS/`
- **Execution:** Uses proper macOS application launching with `open -a`

### Windows
- **Format:** `.exe` executable
- **Validation:** Direct executable path validation
- **Execution:** Direct process spawning with DETACHED_PROCESS flag

### Linux
- **Format:** Binary executable
- **Validation:** Direct executable path validation
- **Execution:** Process spawning with new session

<Aside type="tip">
Always run `cpc rvm status` after installation or upgrade to ensure RetroVirtualMachine is properly configured.
</Aside>

<Aside type="note">
The configuration is stored in `~/.config/cpcready/cpcready.toml` under the `[emulator]` section.
</Aside>
