## Uso

```bash
Uso: cpc emu [OPTIONS]

  Configurar the emulator to use.

  Select between M4Board or RetroVirtualMachine as the default emulator.

Opciones:
  --help  Mostrar this message and exit.
```

## How It Works

1. **Mostrar current emulator** - Shows what's currently configured
2. **Present selection menu** - Interactive list with arrow key navigation
3. **Highlight default** - Current emulator is pre-selected
4. **Guardar choice** - Updates configuration file automatically
5. **Confirm change** - Shows success message with selected emulator

## Navigation

- **‚Üë/‚Üì Arrow keys** - Navigate between options
- **Enter** - Confirm selection
- **ESC/Ctrl+C** - Cancel without changes

<Aside type="tip">
The selected emulator is used by the [`run`](/reference/commands/run/) command when launching software.
</Aside>

<Aside type="note">
After changing the emulator, make sure it's properly configured:
- For RetroVirtualMachine: Use `cpc rvm config` to set the path
- For M4Board: Ensure the board is connected and configured
</Aside>

## Use Cases

### Development Workflow

Switch to RetroVirtualMachine for quick testing and debugging:
```bash
cpc emu  # Select RetroVirtualMachine
cpc run GAME.BAS
```

### Real Hardware Testing

Switch to M4Board for final testing on real hardware:
```bash
cpc emu  # Select M4Board
cpc run GAME.BAS
```

### Multi-Platform Development

Easily test your software on both emulator and real hardware without editing configuration files.
